
# YOLOv8n-TensorRT-Edge-Learning

🚀 **面向 AI 边缘部署的学习与工业级双轨基准测试流水线**

本项目是一个专为 **AI 边缘部署（Edge Deployment）** 学习者设计的实战工程示例。项目没有复杂的业务逻辑，而是将核心聚焦于传统深度学习算法（PyTorch）向工业级边缘端硬件（NVIDIA Jetson / 桌面端 GPU）落地时的**底层优化、格式量化转换（FP16/INT8）与性能基准对齐**流程。

对于想要从“算法研究”迈向“高性能计算与硬件部署”的开发者来说，这是一个极佳的标杆学习与控制变量实验工程。

---

## 🎯 核心学习目标 (Learning Objectives)

通过本项目，你将深入掌握以下 AI 边缘部署的核心技术栈：

1. **硬件级编译与量化校准**：理解 TensorRT 编译器如何重构计算图，掌握 **FP16 半精度** 与 **INT8 极致整型量化（附带小样本校准集）** 的底层差异与算力压榨技巧。
2. **控制变量法基准测试 (Benchmarking)**：学会构建严谨的“双轨制”实验台，直观量化并对比不同精度模型在相同硬件上的吞吐量（Throughput）与延迟抖动（Latency Jitter）。
3. **无头模式 (Headless) 与产物管理**：掌握在纯 SSH 连接的黑盒环境中进行流式推理，并引入**工业级产物版本控制（Artifacts Management）**，实现测试结果按视频源自动隔离归档。
4. **性能观测可视化**：脱离终端日志，学习如何通过代码自动抓取多套实验产物，生成直观的性能对比曲线图谱与量化诊断报告。

本仓库在常规部署流水线的基础上，重构了智能测试与图表分析模块。建议按照以下文件编号循序渐进地进行实验：

## 🎬 核心成果：我们能在这个实验台中看到什么？

在真实的边缘场景中，“模型越来越快”不能只靠直觉，必须依靠严谨的数据支撑。通过依次运行本仓库的代码，你将在 `experiment_results` 目录下收获一份完美的横向对比报告：

1. **算力释放的极限（吞吐量对撞）**：你将亲眼见证，当模型从 FP16 降维到 INT8 运算后，显存占用直接减半，FPS 帧率能迎来 **+50% 甚至更高的爆发式增长**。
2. **发现架构短板（延迟抖动分析）**：通过生成的 `performance_curves.png` 曲线图，你可以直观地看到帧率在画面人流密集时的上下剧烈抖动。这揭示了边缘部署中的典型短板——**由于 CPU 在后处理阶段成为瓶颈所引发的木桶效应**。
3. **“速度 vs 精度”的工程权衡**：通过双屏对比 FP16 与 INT8 产出视频中的检测框，学习者可以深刻理解为什么 INT8 必须依赖量化校准（Calibration），以及如何用肉眼难以察觉的微小精度代价换取海量算力。

## 🛠️ 为什么必须保留 `yolov8n.onnx` 过渡文件？

尽管最终的生产推理只需 `.engine` 文件，但本仓库特意保留了中间过渡格式 `.onnx`。对于部署工程师而言，这是不可或缺的 Debug 与调优利器：

* **算子审查（可视化分析）**：学会使用 **Netron** 等工具打开 `.onnx` 文件，能够直观观察模型的输入输出维度（Dynamic Axes），并检查锚框解码（Box Decoding）等算子是否在框架转换中被正确对齐或发生断裂。
* **脱离 Python 的命令行编译**：保留 ONNX 后，你可以脱离 Python 代码，直接使用英伟达原生的 C++ 命令行工具 `trtexec` 进行最底层的暴力编译与性能压测：

```bash
trtexec --onnx=yolov8n.onnx --saveEngine=yolov8n.engine --fp16 --workspace=4096

```
