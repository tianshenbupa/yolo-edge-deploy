
# YOLOv8n-TensorRT-Edge-Learning

🚀 **面向 AI 边缘部署的学习与高性能实战流水线**

本项目是一个专为 **AI 边缘部署（Edge Deployment）** 学习者设计的实战工程示例。项目没有复杂的业务逻辑，而是将核心聚焦于传统深度学习算法（PyTorch）向工业级边缘端硬件（NVIDIA Jetson / 桌面端 GPU）落地时的**底层优化、格式转换与性能对齐**流程。

对于想要从“算法研究”迈向“高性能计算与硬件部署”的开发者来说，这是一个极佳的标杆学习工程。

---

## 🎯 核心学习目标 (Learning Objectives)

通过本项目，你将深入掌握以下 AI 边缘部署的核心技术栈：

1. **模型中间表示（IR）**：理解 ONNX 的作用，掌握如何导出具备动态尺寸（Dynamic Axes）的模型以适配复杂的边缘端输入。
2. **硬件级编译优化**：理解 TensorRT 编译器如何通过算子融合（Layer Fusion）重构计算图，以及如何开启 **FP16 半精度量化** 榨干硬件算力。
3. **无头模式（Headless Mode）工程实践**：掌握在没有显示器、纯 SSH 连接的边缘盒（如 Jetson 系列）环境中，如何进行健壮的低延迟流式推理。
4. **性能观测与吞吐量量化**：学习如何正确计算并评估边缘端推理的**瞬时 FPS** 与**毫秒级延迟（Latency）**。

## 📂 基于部署流水线的“递进式”仓储架构

本仓库的文件结构就是一条完美的边缘部署学习链路，建议按照文件编号循序渐进地学习：

```text
├── 00-export_yolo.py       # [学习步 1]：通用的 ONNX 格式导出（掌握跨框架解耦与动态输入）
├── 01-convert_trt.py       # [学习步 2]：激活 TensorRT 编译器（掌握硬件专属编译与 FP16 暴力加速）
├── 03-live_inference.py    # [学习步 3]：无弹窗流式推理主程序（掌握 Headless 架构与多流处理）
│
├── yolov8n.pt              # 资产：PyTorch 官方原生权重（部署流水线的源头）
├── test.mp4                # 资产：输入的测试视频流
│
├── yolov8n.onnx            # 中间件：跨平台解耦的通用底座（用于 Netron 拓扑审查）
├── yolov8n.fp16.onnx       # 中间件：编译器底层的量化过渡文件（理解编译期间的精度转换）
├── yolov8n.engine          # 终极产物：针对当前设备专属定制的 TensorRT 硬件加速引擎
│
└── output_trt.mp4          # 核心成果：包含实时性能指标的推理可视化视频

```

## 🎬 核心成果：为什么 `output_trt.mp4` 对边缘端学习至关重要？

在边缘部署的实际工业场景中，设备（如智能相机、车载边缘盒）通常部署在无人值守的黑盒环境中，无法通过 `cv2.imshow` 弹出窗口查看画面。

本仓库最终生成的 **`output_trt.mp4`** 承载了重要的边缘端验证与调优学习价值：

1. **无环境依赖的工程验证**：在纯后台（Headless）运行模式下，该视频证明了整个 TensorRT 推理与 OpenCV 视频流编码流水线在没有图形界面的情况下完全打通。
2. **可视化性能基准（Benchmarking）**：视频左上角实时烙印了 **FPS** 和 **Latency（毫秒级延迟）**。通过观看该视频，你可以直观地评估在开启 TensorRT FP16 加速后，模型在当前硬件上的真实吞吐量和实时响应速度。
3. **结果质量对齐**：通过检查生成的视频，学习者可以量化观察模型从 FP32（.pt）量化到 FP16（.engine）后，目标检测框有无漂移、精度有无肉眼可见的下降，从而深刻理解“精度 vs 速度”的工程权衡。

## 🛠️ 边缘部署进阶核心课题：为什么保留中间过渡文件？

尽管最终的生产推理只需 `.engine` 文件，但本仓库特意保留了 `yolov8n.onnx` 和 `yolov8n.fp16.onnx`。这是部署工程师必须掌握的 Debug 与调优利器：

* **算子审查（可视化）**：学会使用 **Netron** 工具打开 `.onnx`，观察模型的输入输出维度，检查锚框解码（Box Decoding）等算子是否在转换中被正确对齐或发生断裂。
* **脱离 Python 的精细化调优**：保留 ONNX 后，你可以进一步学习如何直接使用英伟达原生的命令行工具 `trtexec`：
```bash
trtexec --onnx=yolov8n.onnx --saveEngine=yolov8n.engine --fp16 --workspace=4096

```


从而掌握如何手动配置显存工作空间（Workspace Size）及多流并发（Streams），这是走向高级 HPC（高性能计算）工程师的必经之路。
